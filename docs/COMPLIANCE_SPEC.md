# 等保合规与数据安全规范

**适用标准**: GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》三级  
**适用系统**: 蓉才通™ HigherMatch ATOS  
**审计周期**: 每季度一次内审，每年一次外审

---

## 1. 安全架构总览

蓉才通系统安全架构基于**零信任原则**设计，核心安全能力由 Trust Layer 可信底座提供，覆盖身份认证、访问控制、数据保护、审计追溯四大领域。

| 安全层 | 能力 | 实现方式 |
|--------|------|----------|
| 身份层 | U盾硬件认证 | 国密SM2签名 + CA网关鉴权 |
| 访问层 | 租户隔离 + RBAC | ScopedToken + 行级数据隔离 |
| 数据层 | 加密 + 脱敏 | SM4传输加密 + k-匿名聚合 |
| 审计层 | 全链路追溯 | SM3哈希存证 + 操作日志 |

## 2. 身份认证安全

### 2.1 U盾硬件登录

系统采用U盾硬件作为唯一登录凭证，不支持用户名/密码登录，从根本上杜绝撞库、钓鱼等攻击向量。

**认证流程**：

1. 客户端检测U盾插入状态
2. 生成随机挑战值（32字节）
3. U盾使用私钥对挑战值进行SM2签名
4. 服务端通过CA网关验证签名有效性
5. 验证通过后颁发ScopedToken（有效期4小时）
6. 全过程SM3哈希存证

**安全措施**：

| 措施 | 说明 |
|------|------|
| PIN码保护 | 3次错误锁定U盾 |
| 证书固定 | 防止中间人攻击 |
| 时钟同步 | ±5分钟容差 |
| 设备指纹 | 异常设备触发二次验证 |

### 2.2 会话管理

| 参数 | 值 | 说明 |
|------|-----|------|
| Token有效期 | 4小时 | 超时强制重新认证 |
| 并发会话数 | 1 | 同一U盾仅允许单设备登录 |
| 空闲超时 | 30分钟 | 无操作自动锁定 |
| Token刷新 | 不支持 | 过期必须重新U盾认证 |

## 3. 访问控制

### 3.1 租户隔离

系统采用**逻辑隔离**方式实现多租户数据隔离，每个企业为独立租户。

**隔离机制**：

- 每次API请求自动注入 `tenantId`（从ScopedToken解析）
- 数据库查询层自动追加 `WHERE tenant_id = ?` 条件
- 跨租户访问一律返回403 + 触发安全告警
- 管理员无法跨租户查看数据（无超级管理员角色）

### 3.2 角色权限矩阵

| 角色 | 岗位管理 | 候选人数据 | 决策审批 | 审计日志 | 系统配置 |
|------|----------|-----------|----------|----------|----------|
| HRD | 读写 | 读写 | 审批 | 只读 | 只读 |
| Recruiter | 读写 | 读写 | 提交 | 无 | 无 |
| Hiring Manager | 只读 | 只读(本岗位) | 审批 | 无 | 无 |
| 候选人 | 只读(公开) | 仅自身 | 无 | 无 | 无 |
| 审计员 | 无 | 无 | 无 | 读写 | 无 |

### 3.3 越权防护

- 前端路由守卫：基于角色的菜单/页面可见性控制
- API网关层：请求级权限校验（ScopedToken + RBAC）
- 数据层：行级安全策略（Row-Level Security）
- 越权尝试：自动记录审计日志 + 实时告警

## 4. 数据保护

### 4.1 数据分类分级

| 级别 | 数据类型 | 保护措施 |
|------|----------|----------|
| L4-机密 | U盾私钥、PIN码 | 硬件安全模块(HSM)存储 |
| L3-敏感 | 身份证号、薪资 | SM4加密存储 + 脱敏展示 |
| L2-内部 | 简历、面试记录 | 访问控制 + 审计日志 |
| L1-公开 | 岗位描述、公司简介 | 无特殊保护 |

### 4.2 脱敏规则

| 数据字段 | 脱敏方式 | 示例 |
|----------|----------|------|
| 手机号 | 中间4位掩码 | 138****5678 |
| 身份证号 | 保留前3后4 | 510***********1234 |
| 姓名 | 保留姓 | 张** |
| 薪资 | 区间化 | 30-40K |
| 邮箱 | 保留域名 | z***@company.com |

### 4.3 k-匿名聚合

任何对外数据聚合（如行业薪资报告、到面率统计）必须满足 **k≥20** 的匿名性要求：

- 聚合样本量 < 20 时，拒绝输出并提示"样本不足"
- 仅展示比例/趋势，不可下钻到个人
- 聚合层统一实现闸门校验

## 5. 审计追溯

### 5.1 审计日志范围

| 操作类型 | 记录内容 | 保留期限 |
|----------|----------|----------|
| 登录/登出 | 时间、IP、设备、结果 | 180天 |
| 数据访问 | 操作人、资源、时间 | 180天 |
| 数据修改 | 操作人、变更前后值 | 365天 |
| 决策审批 | 审批人、决策内容、SM3哈希 | 永久 |
| 安全事件 | 越权、异常访问 | 永久 |

### 5.2 SM3存证

关键操作（决策审批、岗位发布、背书签发）自动生成SM3哈希存证，确保操作不可篡改、可司法举证。

存证格式：`SM3(操作人证书序列号 + 操作类型 + 操作内容 + 时间戳 + 随机盐)`

## 6. 网络安全

| 防护项 | 实现方式 |
|--------|----------|
| DDoS防护 | 云WAF + 速率限制(100 req/s/IP) |
| SQL注入 | 参数化查询 + WAF规则 |
| XSS | CSP策略 + 输出编码 |
| CSRF | SameSite Cookie + Token |
| 文件上传 | 类型白名单 + 病毒扫描 |
