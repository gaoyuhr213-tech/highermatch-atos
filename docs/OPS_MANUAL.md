## 2. 日常运维

### 2.1 健康检查

| 检查项 | 方法 | 频率 | 告警阈值 |

| 前端可用性 | HTTP 200 检查 | 1分钟 | 连续3次失败 |

### 2.2 发布流程

```
代码合并 → CI构建 → 自动化测试 → 灰度发布(10%) → 观察30分钟 → 全量发布
                                                    ↓ (异常)
                                              自动回滚
```

**回滚操作**：

```bash
# 方式1: CDN回源切换至上一版本
aliyun cdn set-origin --domain atos.highermatch.cn --origin oss://highermatch-atos/pc-v2.9/

# 方式2: Vercel回滚
vercel rollback --scope highermatch

# 方式3: 本地回滚
git checkout v2.9.0
pnpm build && deploy
```

## 3. 故障处理

### 3.1 故障分级

| 级别 | 定义 | 响应时间 | 恢复时间 |
|------|------|----------|----------|
| P0 | 系统完全不可用 | 5分钟 | 30分钟 |
| P1 | 核心功能受损 | 15分钟 | 2小时 |
| P2 | 非核心功能异常 | 1小时 | 24小时 |
| P3 | 体验问题 | 4小时 | 下一版本 |

### 3.2 常见故障处理

**故障1: 前端白屏**

```
排查步骤:
1. 检查CDN状态 → aliyun cdn describe-domain
2. 检查源站文件完整性 → curl -I https://atos.highermatch.cn/assets/index-*.js
3. 检查CSP策略 → 浏览器Console是否有CSP报错
4. 清除CDN缓存 → aliyun cdn refresh --url "https://atos.highermatch.cn/*"
```

**故障2: U盾登录失败**

```
排查步骤:
1. 确认U盾驱动已安装 → 检查设备管理器
2. 确认U盾物理连接 → USB设备列表
3. 检查CA网关连通性 → ping ca-gateway.scca.com.cn
4. 检查证书有效期 → 证书管理器查看
5. 查看审计日志 → 确认具体错误码
```

**故障3: 数据加载超时**

```
排查步骤:
1. 检查API网关健康 → curl https://api.highermatch.cn/health
2. 检查数据库连接池 → 监控面板
3. 检查慢查询 → pg_stat_statements
4. 检查Redis缓存命中 → redis-cli info stats
```

## 4. 安全运维

### 4.1 证书管理

| 证书类型 | 有效期 | 续期方式 |
|----------|--------|----------|
| HTTPS证书 | 1年 | 阿里云自动续期 |
| CA网关证书 | 3年 | 四川CA线下续期 |
| 代码签名证书 | 2年 | 手动续期 |

### 4.2 安全巡检

| 巡检项 | 频率 | 工具 |
|--------|------|------|
| 依赖漏洞扫描 | 每日 | `pnpm audit` + Snyk |
| 端口扫描 | 每周 | nmap |
| 渗透测试 | 每季度 | 第三方安全公司 |
| 等保自查 | 每月 | 内部检查清单 |

### 4.3 应急预案

| 场景 | 预案 |
|------|------|
| DDoS攻击 | 启用云WAF高防 + IP黑名单 |
| 数据泄露 | 立即断网 + 通知安全团队 + 取证 |
| 证书泄露 | 立即吊销 + 重新签发 + 全量Token失效 |
| 内部越权 | 冻结账户 + 审计追溯 + 上报 |
