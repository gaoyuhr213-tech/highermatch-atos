# 蓉才通™ HigherMatch — PC端与移动端原型差异说明

> 文档版本: v2.1-Final | 更新日期: 2026-06-12 | 编制: 感知序列 AI+咨询顾问团队

---

## 1. Executive Summary

本文档系统梳理蓉才通™ PC端（桌面Web）与移动端（Expo React Native）原型之间的功能对齐情况、桌面端专属增强、布局适配差异及后续开发建议，确保产品团队与开发团队在跨端实现中保持一致性。

**核心结论：** PC端原型在功能逻辑上与移动端100%对齐，同时针对60万中小微企业台式机办公场景，新增了U盾硬件登录、拖拽看板、右键菜单、批量操作、快捷键、导出等桌面端专属交互。

## 2. 功能对齐矩阵

| 功能模块 | 移动端 | PC端 | 差异说明 |
|---------|--------|------|---------|
| U盾硬件登录 | 无（生物识别替代） | 完整5步流程 | PC端专属：设备检测→PIN→SM2签名→CA鉴权→登录 |
| 决策总控台 | 6指标卡+待决策列表 | 6指标卡+待决策队列+SSE事件流 | PC端增加实时事件流面板、批量审批、快捷键 |
| 招聘流水线 | 纵向滑动Kanban | 6列横向拖拽Kanban | PC端增加DnD拖拽、右键菜单、批量操作、筛选、导出Excel |
| 人才图谱 | 简化2D可视化 | SVG大画布+节点下钻 | PC端增加缩放控件、选中详情面板、hover效果 |
| AI寻访 | 列表+搜索 | 表格+搜索过滤+策略生成 | PC端增加AI策略生成模拟、批量联系、高级过滤 |
| 面试监控 | 对话流 | AI追问模拟+实时评分+SSE | PC端增加多轮追问、维度评分可视化、SSE推送 |
| C端AI教练 | 对话 | 多模式对话+STAR重写 | PC端增加模式切换、STAR框架重写、面试模拟 |
| C端智能投递 | 列表+投递 | 一键/批量投递+状态管理 | PC端增加批量投递、实时状态反馈 |
| C端背书卡片 | 卡片展示 | 完整状态机+QR+验证 | PC端增加状态机流转、QR码模拟、链上验证 |
| 专家端评审 | 任务列表 | 多维评分+CA签名提交 | PC端增加5维度1-10分评分、展开详情、签名存证 |
| 专家端积分 | 积分看板 | 积分看板 | 功能一致 |
| 国企端继任沙盘 | 三列卡片 | 就绪度进度条+风险预警 | PC端增加进度条、详情展开、风险预警 |
| 国企端人才共享 | 列表 | 三列卡片+表格 | 功能一致，布局适配 |
| 决策血统追溯 | Modal | 完整证据链+Agent编排 | PC端增加置信度分解、Agent编排可视化 |
| 角色切换 | Tab内个人中心 | 侧边栏下拉 | 交互形式不同，逻辑一致 |

## 3. PC端专属功能清单

### 3.1 U盾硬件登录（四川CA专属）

PC端独有的国密U盾登录流程，还原四川CA用户真实登录体验：

| 步骤 | 说明 | 技术模拟 |
|------|------|---------|
| Step 1 | U盾设备检测 | 3秒动画模拟USB设备枚举 |
| Step 2 | PIN码输入 | 6位数字密码输入+验证 |
| Step 3 | SM2数字签名 | 进度条模拟国密SM2签名过程 |
| Step 4 | CA身份鉴权 | 模拟CRL/OCSP证书验证 |
| Step 5 | 登录成功 | 显示证书信息+跳转总控台 |

### 3.2 桌面端专属交互

| 交互类型 | 实现位置 | 说明 |
|---------|---------|------|
| 拖拽操作 | 招聘流水线 | @dnd-kit实现候选人阶段推进 |
| 右键菜单 | 招聘流水线 | 右键候选人卡片→快速移至目标阶段/淘汰 |
| 批量操作 | 流水线/寻访/投递 | Checkbox多选+批量动作 |
| 快捷键 | 全局 | Ctrl+L决策血统 / Ctrl+E导出 / ?帮助 |
| 导出功能 | 流水线/继任 | CSV/Excel格式导出 |
| 筛选面板 | 流水线/寻访 | 滑块/下拉过滤 |
| Hover详情 | 图谱/Kanban | 鼠标悬停展示更多信息 |

### 3.3 布局差异

| 维度 | 移动端 | PC端 |
|------|--------|------|
| 导航 | 底部Tab Bar | 左侧固定侧边栏 |
| 信息密度 | 单列纵向滚动 | 多列网格+表格 |
| 分辨率 | 375x812 (iPhone) | 1920x1080 (主流桌面) |
| 交互方式 | 触摸+手势 | 鼠标+键盘 |
| 角色切换 | Tab内个人中心 | 侧边栏顶部下拉 |
| 决策队列 | 纵向卡片列表 | 横向表格+操作按钮 |

## 4. 视觉设计对齐

### 4.1 统一设计Token

| Token | 值 | 用途 |
|-------|---|------|
| Primary | #2563EB | 主色调（AI蓝） |
| Primary Light | #EFF6FF | 背景/hover |
| Background | #F8FAFC | 页面底色 |
| Surface | #FFFFFF | 卡片/面板 |
| Foreground | #0F172A | 主文本 |
| Muted | #64748B | 辅助文本 |
| Border | #E2E8F0 | 分割线 |
| Success | #10B981 | 成功状态 |
| Warning | #F59E0B | 警告状态 |
| Error | #EF4444 | 错误/高风险 |
| Trust | #7C3AED | CA信任色 |

### 4.2 视觉风格定位

对标北美顶级AI SaaS产品视觉语言：

- **Palantir**: 数据密度与信息层级
- **Linear**: 简洁克制的交互设计
- **Vercel**: 明亮通透的视觉基调
- **Anthropic**: AI对话界面设计

具体实现：白底为主，避免暗色系（适配企业办公场景）；Glassmorphism卡片（backdrop-blur + 半透明边框）；渐变色仅用于品牌标识与关键CTA；Inter字体系统，层级分明的排版。

## 5. 技术栈对比

| 维度 | 移动端 | PC端 |
|------|--------|------|
| 框架 | Expo SDK 54 + React Native | React 19 + Vite 8 |
| 样式 | NativeWind (Tailwind) | Tailwind CSS 3.4 |
| 路由 | Expo Router 6 | React Router 6 |
| 状态 | React Context | React Context (同构) |
| 拖拽 | - | @dnd-kit/core |
| 图标 | SF Symbols / MaterialIcons | Lucide React |
| 图表 | - | Recharts |
| 动画 | Reanimated 4 | Framer Motion |
| 部署 | Expo Go / APK | 静态站点 (Vercel/Netlify/OSS) |

## 6. 演示操作手册

### 6.1 本地启动

```bash
# 安装依赖
cd highermatch-pc && npm install

# 开发模式
npm run dev

# 构建生产版本
npm run build

# 预览生产版本
npm run preview
```

### 6.2 部署方式

**方式一：静态托管（推荐）**

将 `dist/` 目录部署到 Vercel / Netlify / 阿里云OSS / 腾讯云COS 即可。

**方式二：本地演示**

```bash
npm run build
npx serve dist -l 4173
# 浏览器打开 http://localhost:4173
```

### 6.3 演示路径

1. **U盾登录流程**: 打开首页 → 等待设备检测(3s) → 输入任意6位PIN → 观察签名动画 → 进入总控台
2. **决策总控台**: 查看6大指标 → 审批待决策 → 点击"决策血统"查看追溯
3. **招聘流水线**: 拖拽候选人卡片 → 右键快捷操作 → 批量选择 → 导出Excel
4. **角色切换**: 侧边栏顶部下拉 → 选择C端/专家端/国企端
5. **AI面试**: 面试监控 → 查看追问对话 → 观察实时评分
6. **背书验证**: C端背书卡片 → 点击"发起背书" → 观察状态流转
